实验目的

通过实践操作，掌握基于云计算环境的网站部署技能。具体包括：构建环境并实现个人博客/网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客 / 网站上。

实验环境

硬件环境： 联网的计算机一台

软件环境： Windows 11 + Git + GitHub 账号 / Linux (Ubuntu) 均可

关键工具： GitHub Pages、Markdown、Git 命令行

实验步骤与内容

本次实验利用 GitHub 平台提供的 GitHub Pages 服务，免费搭建个人博客网站，并发布第一篇技术文章（即本实验报告）。

  • 1. 登录 GitHub 账号并创建仓库
    仓库名严格按照格式命名为 chehaixia.github.io，这是 GitHub Pages 的特殊命名规则，系统会自动识别并生成个人站点的域名。
  • 2. 初始化仓库并添加 README 文件
    创建仓库时勾选 “Add a README”，自动初始化 main 分支，确保仓库拥有可发布的内容。
  • 3. 配置 GitHub Pages 来源
    进入仓库 Settings → Pages，将 “Source” 选为 “Deploy from a branch”，分支选择 main，点击保存。大约 1~2 分钟后站点即可通过 https://chehaixia.github.io 访问。
  • 4. 准备实验报告内容
    将实验报告保存为 Markdown 格式（.md），并把报告中引用的图片上传到仓库的 images 文件夹，保证图片路径正确，GitHub 能完整渲染出带插图的文章。
  • 5. 提交并部署文章
    将修改后的 .md 文件以及 images 文件夹通过 git add .、git commit、git push 推送到远程仓库，GitHub Actions 会自动部署更新。
  • 6. 在线访问与验证
    在浏览器输入 https://chehaixia.github.io 即可看到个人博客主页，点击对应文章链接即可阅读完整实验报告。
# 常用 Git 推送命令示例
git init
git add .
git commit -m "发布实验报告博客"
git remote add origin https://github.com/chehaixia/chehaixia.github.io.git
git push -u origin main
结论分析与体会

一、实验结论
利用 GitHub Pages 可免费搭建个人静态博客，无需购买服务器与域名。核心流程为：创建 username.github.io 仓库 → 初始化 main 分支 → 配置 Pages → 编写发布文章 → 提交部署 → 在线访问。文章采用 Markdown 格式编写，GitHub 自动渲染为网页，排版清晰，支持插入代码块、图片及列表等丰富样式。仓库命名与分支配置正确是博客成功上线的关键。

二、实验体会
本次实验让我深入掌握了 GitHub 仓库管理与 Pages 部署的基本操作，理解了静态网站从代码到访问的完整发布流程。在配置过程中曾遇到分支未生效、图片路径错误等问题，通过查阅文档和逐步排查，最终成功上线。这个过程极大提升了我的问题解决能力与对 Git 工作流的熟练度。

同时，我也认识到 GitHub 不仅是代码托管平台，更是知识整理与技术分享的优质工具。未来我会持续用它记录学习笔记、项目文档，逐步完善个人技术博客体系。
