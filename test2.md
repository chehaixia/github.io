实验步骤与内容：
利用GitHub平台创建自己的博客，发布文章。
登陆自己的GitHub账号，创建新仓库，将仓库的名字命名成chehaixia.github.io
![p1](/images/p1.png)

并且create a new file，添加一个简单的README.md文件，自动创建新的分支main
![p2](/images/p2.png)


在仓库里settings里面找到pages页，source选Deploy from a branch，branch选刚刚创建的main分支，save之后准备好要发布的文章。
将本实验报告在存储的时候后缀名改为.md，然后将实验报告里面的图片也传到GitHub仓库里新建的images文件夹里，这样GitHub可以完整加载整个报告。











结论分析与体会：
利用GitHub Pages可免费搭建个人静态博客，无需服务器与域名；核心流程是创建usename.github.io 仓库、初始化 main 分支、配置 Pages、编写发布文章、提交部署、在线访问。文章以 Markdown 格式编写，GitHub 自动渲染为网页，支持丰富排版。仓库命名与分支配置正确是博客成功上线的关键。
本次实验让我掌握了 GitHub 仓库管理与 Pages 部署的基本操作，理解了静态网站的发布流程。遇到问题时的排查过程，提升了我的问题解决能力。同时，我也认识到 GitHub 不仅是代码托管平台，更是知识整理与技术分享的优质工具，未来我会持续用它记录学习内容，完善个人知识体系。
